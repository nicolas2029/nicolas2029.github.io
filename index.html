Tecnologías

En esta sección se muestran las principales tecnologías que domino relacionadas al desarrollo de aplicaciones web.

Golang

Mi lenguaje de programación principal, con el cual desarrolló microservicios siguiendo los principios de desarrollo SOLID mediante programación orientada a objetos utilizando diferentes patrones de diseño y asegurando el funcionamiento de los servicios mediante el desarrollo guiado por pruebas. Principalmente utilizo peticiones de tipo REST y GRPC.

PostgreSQL

Es el principal motor de base de datos que manejo en donde administro diferentes usuarios y bases de datos. Siempre doy importancia al diseño de una DB, para esto normalizo todas mis tablas y busco la mejor manera de modelar el sistema brindando una mejor solución implementando queries que ofrecen un alto rendimiento.

Docker y Kubernetes

Docker permite automatizar el despliegue de aplicaciones creando contenedores, haciendo uso de kubernetes podemos administrar dichos contenedores, lo cual nos permite escalar y dar una mayor tolerancia a fallas, también nos permite desplegar las aplicaciones en diferentes servidores distribuidos, ya sean privados o en la nube.

MongoDB

Tengo ciertos conocimientos y experiencia utilizando bases de datos NoSQL como MongoDB, hay ocasiones en donde prefiero utilizar este tipo de bases de datos como pueden ser en sistemas donde se quieran modelar datos como productos en donde los precios suelen cambiar seguidamente o al manejar direcciones prefiero utilizar Bases de datos NoSQL.

Git y Github Actions

Git permite llevar el control de versiones de la aplicación, además de poder trabajar en equipo de forma remota y asíncrona, lo cual acelera el proceso de desarrollo. Utilizo repositorios en Github tanto públicos como privados en donde implementó las acciones de Github para desplegar aplicaciones en el servidor de pruebas y/o de producción de forma automática.

Otros

Además de las herramientas ya vistas, también utilizo algunas otras como Postman para testear y documentar las apis, Jira para asignar actividades y llevar un control del proceso de desarrollo de forma transparente, StarUML / draw.io para diseñar los diagramas del sistema, HTML, CSS, JS, Bootstrap, un poco de Angular y conocimientos básicos de React.
